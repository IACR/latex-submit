This site is under development and is not yet ready to be used. It is here only for testing purposes.

This site is intended to become part of the IACR publishing pipeline, starting with the IACR Communications in Cryptology. Our goal is to reduce the cost of open access publishing by reducing the amount of human effort required by both authors and editors in the production of a paper. Once a paper is accepted, authors will be directed to this site in order to submit their final versions in LaTeX format. Their sources will be compiled, and after some light copy editing, the paper will be ready for publication within seconds rather than weeks or months.

Papers submitted to the IACR Communications in Cryptology will be required to use the iacrcc document class. At the present time we also support uploads for other publishing venues, but the main purpose of this site is to support a new generation of LaTeX document classes. The approach for this journal is described in this paper.

The purpose of this system is:

  1. Safely validate that the LaTeX compiles correctly under the new iacrcc document class.
  2. Validate that the author has supplied sufficient metadata (e.g., title, affiliations, funding information, etc).
  3. Assign a DOI immediately upon submission.

Automated submission systems like this are a bit complicated to run because it is generally unsafe to accept LaTeX sources from an untrusted source. This system does not support all of the thousands of LaTeX packages. The list of supported packages is given below, along with some packages that are explicitly forbidden.
